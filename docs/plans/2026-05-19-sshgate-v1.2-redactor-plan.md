# SSHGate v1.2 Output Redactor — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> Each task is sized for one subagent dispatch (≤ 6h focused work). The subagent applies strict TDD inside each task: write failing test → confirm red → minimal code → confirm green → refactor → commit. The subagent reports back with a ≤200-word summary and the commit SHA.

**Goal:** Ship the v1.2 output redactor — three detection layers (named-format regex, file-mode heuristic, operator redactlist/unredactlist) plus a depth-1 base64/hex/URL decode pass, plus a `SSHGATE_CMD:` wire-envelope namespace with 11 meta-commands. Bytes flow `child stdout/stderr → redact.Writer → SSH pipe → MCP → agent`. The agent never sees pre-redaction bytes.

**Operator framing — simplest version possible, quickly.** This v1.2 ships ONLY the `standard` mode (Layer 1 named regex + Layer 2 file-mode + Layer 3 redactlist + depth-1 decode). `thorough` mode, verified mode, Landlock kernel enforcement, BPE token-efficiency, and per-host tokenizer deps are stubs / `// TODO(v1.2.1)` and tracked in `docs/FUTURE.md`. No optionality knobs the operator does not need.

**Architecture summary:**
- **Where it lives:** inside the gate binary on the remote host, between the child process's stdout/stderr and the SSH stream. The MCP-side trust boundary is bypassable; the gate is the only physical choke point.
- **Per-process == per-session:** OpenSSH spawns a fresh `gate` per forced-command. The 32-byte HMAC salt is generated by `crypto/rand` at startup and never persisted.
- **Vendored rules, not imported:** gitleaks-derived named-format rules are extracted to Go source under `src/redact/rules/gitleaks/`; SSHGate-native rules under `src/redact/rules/sshgate/`. `go generate` emits a committed `rules_combined.go`.
- **Marker format:** `[SSHGATE_REDACTED key=<8hex>]` for inline matches, `[SSHGATE_REDACTED_FILE path=<p> mode=<oct> sha256=<8hex>]` for whole-file. No `via=`, no source info inline. Provenance via signed `redact.why`.
- **Two append-only files:** `redactlist.append-only` (positive entries) and `unredactlist.append-only` (negative entries / `unmask:` overrides). Tombstone-based removal. Per-entry JSON metadata.
- **Wire-envelope:** `SSHGATE_CMD:<sig>:<payload>` JSON; 11 subcommands; sign-requirement matrix (REDUCING redaction needs sign, INCREASING does not).

**Tech Stack:** Go 1.22+; stdlib `crypto/hmac`, `crypto/sha256`, `crypto/rand`, `crypto/ed25519`, `encoding/json`, `regexp`. No new third-party dependencies. Test stack unchanged from v1.

**Spec reference:** `docs/audits/secrets-redaction-architecture-2026-05-19.md` — every task implements a locked section of that document. Subagents MUST read the relevant section before starting.

**Code quality rubric:** `~/arogara/code-review/guidelines/{general,go,daemon}.md`. Subagents implement to this standard; deviations get flagged in the commit message.

**Dependencies on prior work:**
- v1.0 (shipped): `src/sigwire` (envelope), `src/gate/{verify,keystore,executor}.go`, `src/classify`.
- v1.1 (parallel): `SSHGATE_UPDATE` implementation lands separately. This plan assumes update is either shipped or stubbed.

package redact

type Rule struct {
    ID           string
    Description  string
    Regex        *regexp.Regexp

// NewWriter wraps w; every byte written passes through Layer 1 scan (rules)
// using sessionSalt for HMAC marker keys. dst receives the scrubbed stream.

**Interface contract:**
```go
package redact

// Entry is one append-only line in redactlist or unredactlist.
type Entry struct {
    RuleID    string    `json:"rule_id"`
    Kind      string    `json:"kind"`        // "pattern" | "anchor" | "file" | "unmask" | "tombstone"
    Value     string    `json:"value"`
    AddedAt   time.Time `json:"added_at"`
    AddedVia  string    `json:"added_via"`   // "redact.add" | "unredact.add" | "redact.remove" | …
    Signed    bool      `json:"signed"`
    SessionFP string    `json:"session_fp"`
    AgentHint string    `json:"agent_hint,omitempty"`
    Supersedes string   `json:"supersedes,omitempty"`
    Sig       []byte    `json:"sig,omitempty"`  // ed25519 sig over canonical-JSON of the above
}

// Store holds the parsed net set after tombstone walk.
type Store struct {
    Patterns []*regexp.Regexp
    Anchors  map[[32]byte]struct{}  // HMAC keys
    Files    map[string]struct{}
    Unmasks  map[string]struct{}    // unredact-only
}

// Load reads both files from baseDir, validates every signature against
// pubkey, applies tombstones, returns the net Store. Returns an error
// (fail-closed) on any tampered line.
func Load(baseDir string, pubkey ed25519.PublicKey) (redact, unredact *Store, err error)

// AutoValidate runs the 6-step pipeline. Returns the rejection reason
// on failure or nil on success.
func AutoValidate(pattern string) error

// Envelope is the parsed SSHGATE_CMD wire form.
type Envelope struct {
    Sig     []byte           // empty for unsigned
    Payload struct {
        Cmd  string                 `json:"cmd"`
        Args map[string]any         `json:"args"`
    }
}

func ParseEnvelope(line string) (Envelope, error)
```

**Conflict-resolution order (lifted from spec §"Conflict resolution"; the loader and scanner must honour this exactly):**

1. Layer 1 named regex (built-in + sshgate-native + gitleaks-vendored). Matches are **sticky** — only a signed `redact.remove` of that rule un-sticks. Unredactlist entries do NOT cancel Layer 1 named hits.
2. Layer 2 file-mode heuristic. Consult unredactlist `unmask:` entries on the file path; remove matching marks.
3. Layer 1 entropy (thorough only — absent in v1.2). Consult unredactlist; remove matching.
4. Recursive decode pass. Consult unredactlist; remove matching.
5. Layer 3 redactlist (user-curated). Sticky — only signed `redact.remove` un-sticks.
6. Emit remaining flagged spans as redactions.

This ordering is a hot-path invariant; document it at the top of `scanner.go` and reference the spec section by number.

- [ ] **Step 8:** Write `envelope_test.go`: `SSHGATE_CMD::<payload>` (unsigned), `SSHGATE_CMD:<sig>:<payload>` (signed), bad base64 in sig, malformed JSON payload, backwards-compat alias for `SSHGATE_REVOKE` / `SSHGATE_UPDATE` → translated to `{"cmd":"revoke"}` / `{"cmd":"update"}`.

### Task R3 [parallel with R4]: File-mode heuristic with predicate registry

**Spec section:** §"Layer 2 — file-mode heuristic (registry of predicates)"

**Files:**
- New: `src/redact/filemode/{doc,parse,predicates,registry}.go` + tests
- Modified: `src/redact/scanner.go` (hook in pre-exec call to filemode predicates)

**Interface contract:**
```go
package filemode

// Top-of-file comment: "Layer 2 is a rat race. New bypasses become new
// predicates. See FUTURE.md for kernel-level read enforcement (Landlock)."

type ParsedCmd struct {
    Argv      []string
    Pipeline  [][]string  // each stage's argv; len==1 for non-pipelines
    Redirects []Redirect  // `<` redirects
    BailOut   bool        // true when shell metacharacters fired
}

type SensitiveFileMatch struct {
    Path   string
    Mode   uint32
    SHA256 [8]byte  // truncated; for whole-file marker
    Reason string   // which predicate fired
}

// Parse returns ParsedCmd or BailOut=true if any shell metacharacter is present.
func Parse(cmd string) ParsedCmd

// Check runs every predicate against parsed in registry order; returns the
// union of matches. Consults unmasks for path overrides.
func Check(parsed ParsedCmd, unmasks map[string]struct{}) []SensitiveFileMatch
```

**Required predicates (the v1.2 floor — spec §"Layer 2 → Required predicates"):**

| # | Predicate | Triggers on | Emits |
|---|---|---|---|
| 1 | `catSingleFile` | `cat <one-file>`, mode `& 0o077 == 0` | whole-file marker |
| 2 | `catMultiFile` | `cat file1 file2 file3` — per-file mode check | one marker per 0600 file |
| 3 | `pipelineFirstStageReads` | pipelines where stage 0 reads a 0600 file (e.g. `cat /etc/shadow \| head -1`, `tail -f /etc/shadow \| grep root`) | whole-output marker |
| 4 | `readerOverFile` | `grep`, `head`, `tail`, `less`, `bat`, `nl`, `more` over a 0600 file as a positional arg | whole-output marker |
| 5 | `inputRedirect` | `command < /etc/shadow`, `read x < /etc/shadow` | whole-output marker |

Predicates run in registry order; first match wins per file path. Each match consults `unredactlist.append-only` for a matching `unmask:` entry — if found, the heuristic mark is suppressed for this command only (the rule is NOT removed from the registry).

**Known unhandled bypasses (document in `doc.go`, do NOT try to fix in v1.2):**

- 0644 config files with embedded secrets (Layer 1 + Layer 3 must catch).
- Command substitution (`$(cat /etc/shadow)`), backticks, eval-wrapped reads → metacharacter-guard bails out by design.
- Network-mediated reads: `curl file:///etc/shadow`, `nc -l < /etc/shadow`.
- Tools not enumerated: `xxd`, `od`, `hexdump`, `strings`, `awk -f`, `sed -n p`, `python -c "open(...)"`.
- `sudo cat /etc/shadow` where the SSH user has NOPASSWD sudo.

These appear as v1.2.1+ predicate additions in `docs/FUTURE.md`.

**Subtasks:**
- [ ] **Step 1:** Write `parse_test.go`: tokenizer, pipeline detection, redirect detection; metacharacter list (`$`, backtick, `;`, `>`, `&`, `*`, `?`, `~`, `(`, `)`, `{`, `}`) → `BailOut=true`.
- [ ] **Step 2:** RED → implement `parse.go`. Use a tiny shell tokenizer; do NOT call `/bin/sh -n`.
- [ ] **Step 3:** Write `predicates_test.go`: per-predicate temp-file fixtures (mode 0600 / 0644 / 0640) in `t.TempDir()`; verify each predicate fires only on 0600; verify unmasks suppress whole-file markers.
- [ ] **Step 4:** RED → implement `predicates.go` + `registry.go`.
- [ ] **Step 5:** Hook into `scanner.go`: at write start, parse the inbound command once; if filemode matches, prepend the `[SSHGATE_REDACTED_FILE …]` marker(s); skip Layer 1 stream scan for that command (whole-output is already replaced).
- [ ] **Step 6:** `go test -race ./src/redact/filemode/...` clean.
- [ ] **Step 7:** Commit: `feat(redact/filemode): predicate registry with 5 v1.2 predicates (task R3)`

**Complexity:** Medium. ~300 LOC production + ~450 LOC test. **Estimated: 1 day.**

- `cat $(echo /etc/shadow)` → BailOut → Layer 1 only.

### Task R4 [parallel with R3]: Recursive decode pass (depth 1)

**Spec section:** §"Recursive decode pass"

**Files:**
- New: `src/redact/decode.go` + tests

**Interface contract:**
```go
// View is a decoded segment plus the byte range in the original buffer.
type View struct {
    Bytes     []byte
    OrigRange [2]int  // [start, end) in the original buffer
}

// Views returns 1..N decoded views of buf at the given depth.
// v1.2 standard mode calls with depth=1. depth=3 is reserved for
// thorough mode (// TODO(v1.2.1)).
func Views(buf []byte, depth int) []View
```

### Marker examples (the canonical wire forms — for fixtures and golden tests)

```
Inline match (Layer 1, decode pass, or Layer 3 pattern/anchor):
    [SSHGATE_REDACTED key=abc12345]

Whole-file (Layer 2 file-mode predicate or Layer 3 file entry):
    [SSHGATE_REDACTED_FILE path=/etc/ssh/ssh_host_ed25519_key mode=0600 sha256=def67890]
```

- `key` = first 32 bits (8 hex chars) of `HMAC-SHA256(per-session-32-byte-salt, matched-bytes)`. Same secret in the same session → same key. New session → fresh salt → different key for the same secret.
- `sha256` on the file marker = first 32 bits of `SHA-256(file contents)`, truncated to 8 hex chars. Lets the agent recognise "same file as before" within a session without learning what's in it.
- No `via=` field. No source/reason inline. Provenance is via the signed `redact.why <key>` command only.
- Both stdout and stderr pass through their own `redact.Writer`. Stderr is treated identically to stdout.

## Rollout and migration (lifted from spec §"Rollout and migration")

1. **v1.2.0 ships with `standard` as the default** for both fresh installs and upgrades. There is no `fast` mode to inherit; `standard`'s aggressive optimisation makes it the universal default.
2. **`SSHGATE_UPDATE`** (v1.1) is the deployment vehicle. Operators roll back v1.2 → v1.1 by signing an `SSHGATE_UPDATE` to the previous binary.
3. **Schema versioning of the append-only files**: v1.2 gate refuses to start if either file exists with an unrecognised schema version (daemon guideline 5.5). v1.2.0 schema is `"v1"`.
4. **There is no opt-out flag.** If an operator wants no redaction, they uninstall v1.2 and roll back to v1.1. Vault's anti-`log_raw` discipline drives this — a redaction-off flag inevitably ships to production by mistake.
5. **Backwards-compat for `SSHGATE_REVOKE` / `SSHGATE_UPDATE`**: both old wire forms accepted for one release alongside the `SSHGATE_CMD:` envelope; deprecation warning logged.
